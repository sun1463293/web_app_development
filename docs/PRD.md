# 產品需求文件 (PRD)：個人記帳系統

## 1. 專案概述

### 背景與動機
現代人常面臨不知道錢花去哪裡的困擾，導致無法有效儲蓄與理財。為了幫助個人更好地管理財務狀況，我們需要一個簡單易用、能清楚呈現收支狀況的個人記帳系統，讓使用者能輕鬆追蹤每日開銷、掌握可用餘額，並了解自身消費習慣。

### 目標用戶
- 各種需要記帳的人（包括學生、上班族、家庭主婦等）
- 想要了解自己每月花費比例、希望改善消費習慣的人
- 需要記錄儲蓄進度的人

### 核心價值主張
提供直覺的記帳體驗與清晰的圖表分析，幫助使用者輕鬆掌握財務狀況，達成財務目標。

## 2. 功能需求

以下為系統的主要功能及對應的使用者故事：

### 1. 記錄收入
- 作為 **記帳使用者**，我希望 **能記錄每筆收入（如薪水、獎金等）**，以便 **知道我總共有多少錢可以分配**。

### 2. 記錄支出與分類
- 作為 **記帳使用者**，我希望 **能記錄每筆支出，並將其分類為食、衣、住、行、育、樂等類別**，以便 **追蹤我把錢花在哪些地方**。

### 3. 記錄儲蓄
- 作為 **記帳使用者**，我希望 **能單獨記錄我的儲蓄金額**，以便 **追蹤我的存錢進度**。

### 4. 計算可用餘額
- 作為 **記帳使用者**，我希望 **系統能自動計算總收入減去總支出與儲蓄後的可用餘額**，以便 **隨時掌握我還有多少錢可以花**。

### 5. 查看每月花費比例
- 作為 **記帳使用者**，我希望 **能查看每個月在食、衣、住、行、育、樂等各類別的花費比例圖表**，以便 **了解我的消費結構並找出可以節省的項目**。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask (Python)
- **前端樣板**：使用 Jinja2
- **資料庫**：使用 SQLite (輕量級關聯式資料庫)

### 效能與安全考量
- **效能**：系統操作（如新增收支紀錄、載入圖表）應保持流暢，頁面載入時間應小於 2 秒。
- **安全**：確保使用者資料的隱私，避免 SQL Injection 等基本網路攻擊。若未來加入多使用者系統，需實作帳號密碼雜湊與 Session 管理。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須要有)
- 記錄收入與支出（包含金額、日期、備註）。
- 支出分類功能（食、衣、住、行、育、樂）。
- 記錄儲蓄功能。
- 自動計算當前可用餘額（總收入 - 總支出 - 儲蓄）。

### Should Have (應該要有)
- 每月各類別支出比例的圓餅圖或長條圖。
- 查詢歷史收支紀錄的列表。

### Nice to Have (有了更好)
- 預算設定與超支提醒功能。
- 匯出為 CSV 或 Excel 檔案的功能。
- 支援多帳戶（例如：現金、信用卡、銀行帳戶）管理。
